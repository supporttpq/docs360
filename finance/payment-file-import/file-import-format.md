# File Import Format

## PBS Payment File Format & Import Documentation

_FI\_payments\_17\_03\_2026.PBS · 132 payment records · Total: 687,522.00_

## 1. File Overview

<figure><img src="../../.gitbook/assets/image (759).png" alt=""><figcaption></figcaption></figure>

A PBS file is a fixed-width ASCII text file produced by the PBS (Pengeinstitutternes BetalingsService) Danish payment clearing system. Each line is a fixed-length record identified by a 4-character record type prefix. Lines are terminated with CR+LF (Windows line endings).

This specific file — FI\_payments\_17\_03\_2026.PBS — contains 136 lines total and represents payments collected on behalf of creditor 45801012 and processed on 17 March 2026.

## 3. FI02 — Batch Header

<table data-header-hidden><thead><tr><th width="134"></th><th></th></tr></thead><tbody><tr><td>FI02</td><td>Batch Header · Trimmed length: 65 characters · Appears once, second line</td></tr></tbody></table>

The FI02 record describes the payment batch. The C# import code uses this record to verify that the file belongs to the correct creditor — if the creditor number extracted from this line does not match the system's expected value, the import is rejected with an error.

#### Creditor validation logic (C#)

```csharp
if (line.Trim().Length == 65) {
    string creditorNo = line.Substring(5, 8);
    if (importPaymentsInformation.CreditorNumber != creditorNo) {
        errorMessage = "The creditor number from the file does not correspond with the creditor number of your agency";
        break;
    }
}
```

**Example line:**

```
FI020820402267780000207443300000000000000000000000020260324112701
```

## 4. FI03 — Payment Record

<table data-header-hidden><thead><tr><th width="112.8887939453125"></th><th></th></tr></thead><tbody><tr><td>FI03</td><td>Payment Record · Trimmed length: 113 characters · Repeats once per payment (132 records)</td></tr></tbody></table>

FI03 is the core data record. Each line represents a single payment collected from a payer (subscriber) and credited to the agency. The C# importer reads every FI03 line and constructs an `ImportedPayment` object from six fields within it.

#### Import code (C#)

```csharp
else if (line.Trim().Length == 113) {
    dateCustomerPaidString = line.Substring(5, 8);
    payerID = line.Substring(19, 15);
    bookingNoText = line.Substring(19, 12);
    accountingDateString = line.Substring(56, 8);
    trackingCode = line.Substring(46, 10);
    ammountString = line.Substring(64, 15);
    bookingNo = Int64.Parse(bookingNoText);
    amount = int.Parse(ammountString) / 100; // stored in øre (cents)
    importedPayments.Add(new ImportedPayment {
        AccountingDate = accountingDate,
        Amount = amount,
        BookingNo = bookingNo,
        DateCustomerPaid = dateCustomerPaid,
        PayerID = payerID,
        TrackingCode = trackingCode
    });
}
```

**Example line:**

```
FI030202603167100000000007745180132225202603163573674200202603170000000000996000000000000000000099600CC000000000N
```

{% hint style="info" %}
Note on overlapping fields: PayerID (positions 19–34, 15 chars) and BookingNo (positions 19–31, 12 chars) share the same starting position. BookingNo is simply the first 12 characters of PayerID. The C# code reads both independently using different `Substring()` lengths.
{% endhint %}

{% hint style="info" %}
Note on amount encoding: amounts are stored as integers in the smallest currency unit (øre for DKK, eurocents for EUR). The value 000000000099600 = 99,600 øre = 996.00 DKK. Always divide the raw field by 100 to obtain the decimal amount.
{% endhint %}

## 5. FI08 — Batch Summary

<table data-header-hidden><thead><tr><th width="109.5555419921875"></th><th></th></tr></thead><tbody><tr><td>FI08</td><td>Batch Summary · Trimmed length: 53 characters · Appears once after last FI03</td></tr></tbody></table>

The FI08 record closes the payment batch. It provides control totals that can be used to verify the integrity of all FI03 records: the total record count and the grand total of all amounts. The C# importer does not currently read this record, but it should be used for reconciliation.

**Example line:**

```
FI0808204022600000000132000000068752200000000000000000
```

## 6. FI09 — File Trailer

<table data-header-hidden><thead><tr><th width="112.888916015625"></th><th></th></tr></thead><tbody><tr><td>FI09</td><td>File Trailer · Trimmed length: 53 characters · Appears once, last line</td></tr></tbody></table>

The FI09 record is the file-level footer. It repeats the creditor identification from FI01 and provides a final record count covering the entire file. It can be used as a second-level integrity check, independent of the batch summary in FI08.

**Example line:**

```

```

## 9. File Integrity Checks

The PBS format provides several built-in controls that can be used to validate a file before or after import:

<table><thead><tr><th width="183.4444580078125">Check</th><th width="238.6666259765625">Source Fields</th><th>Validation Rule</th></tr></thead><tbody><tr><td><strong>Creditor Number</strong></td><td>FI01[5:13] vs FI09[5:13]</td><td>Both must match. If they differ, the file may be corrupted or belong to a different creditor.</td></tr><tr><td><strong>Creditor vs System</strong></td><td>FI02[5:13] vs stored value</td><td>Enforced by C# importer. Import aborts if mismatch detected.</td></tr><tr><td><strong>Record Count</strong></td><td>FI08 count vs FI09 count</td><td>Both should equal 132 (number of FI03 lines). Count actual FI03 lines and compare.</td></tr><tr><td><strong>Grand Total</strong></td><td>FI08 GrandTotal vs Σ FI03</td><td>Sum of all FI03 Amount fields (raw int) must equal the GrandTotal in FI08 (68,752,200).</td></tr><tr><td><strong>Running Total</strong></td><td>FI03[93:101] (per line)</td><td>Each FI03 line contains a running cumulative total. The last FI03 running total must equal the FI08 GrandTotal.</td></tr><tr><td><strong>File Date vs Name</strong></td><td>FI01[43:51] vs filename date</td><td>The filename FI_payments_17_03_2026 implies 17 March 2026. The FI01 creation date is 2026-03-24 (transmission date, not collection date).</td></tr></tbody></table>
