# What is Tourpaq?

### Overview

**Tourpaq** is a travel management platform built specifically for **tour operators**. It helps you manage the full lifecycle of a travel product — from building price lists and hotel contracts, to taking bookings, handling payments, communicating with customers, and exporting data to suppliers.

Think of it as the central hub that connects your **products**, **customers**, **bookings**, and **finances** in one place.

***

### What can you do in Tourpaq?

| Area               | What it covers                                               |
| ------------------ | ------------------------------------------------------------ |
| 📋 **Bookings**    | Create, edit, and manage customer reservations               |
| 🏨 **Hotels**      | Set up hotels, room types, contracts, and availability       |
| ✈️ **Transport**   | Configure flights, buses, routes, and seating                |
| 💰 **Finance**     | Register payments, manage refunds, and track profitability   |
| 👤 **Customers**   | Manage customer profiles and communication history           |
| 📊 **Dashboard**   | Get a real-time snapshot of sales, bookings, and performance |
| 📧 **Email & SMS** | Send automated and manual communications to customers        |
| 🌍 **Guest App**   | Manage destination content for the mobile guest experience   |
| ⚙️ **Setup**       | Configure system settings, brands, users, and integrations   |

***

### How is Tourpaq structured?

Tourpaq is organised around **Brands**. A Brand represents a distinct travel company or sales channel (e.g. different tour operator labels under the same group). Most data — price lists, hotels, transports, bookings — is linked to a specific Brand.

{% hint style="warning" %}
**Always check your Brand** before creating or searching for data. If you're looking at the wrong Brand, you won't see the right hotels, prices, or bookings.&#x20;
{% endhint %}

***

### Who uses Tourpaq?

Tourpaq is used by several different roles within a travel organisation:

* **Booking Agents** — Create bookings, manage customers, handle communication
* **System Administrators** — Configure the system, manage users and brands, set up integrations
* **Finance Teams** — Register payments, handle refunds, monitor profitability
* **Hotel Managers / Suppliers** — View contracts, availability, and confirmations

Each role uses a different part of the system. See Your Learning Path by Role to find the pages most relevant to you.
